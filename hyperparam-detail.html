走向工业化交付：Agent 评测平台与双端成本引擎闭环

2025.12 - 2026.03 超参数科技 (Knit 项目) 技术型 AI 产品经理 (AIPM)
Knit 平台 v1.0：面向 Agent 工业化交付的评测与治理看板
核心定位：在实习期间，我承担了偏“技术型 AIPM”的复合角色。一边从产品维度定义信息架构与交互范式（Agent-Driven UI），一边下场编写 Python 引擎与构建 JSON 数据库，用代码打通业务的最后一公里。

我将工作对象严格聚焦在“AI 工作流与 Agent 的工业化交付”上，致力于解决大模型落地过程中的三大痛点：可测（EvalFlow）、可诊断（Trace 归因）、可算账（模型与基建双端 ROI）。

01 / 产品架构：EvalFlow 评测平台与体验重构

面对传统大模型“测试靠手工、排错靠猜”的窘境，我反向梳理并沉淀了 EvalFlow 的完整信息架构与核心术语体系，落地了可执行的模块化方案。

  • 完整信息架构：确立了“评测集生成/审核 → 跑批/门禁 → A/B 对撞 → 诊断修复 → 成本管理”的黄金业务链路。
  • 创新交互范式：定义并固化了 Agent-Driven UI，让 Copilot 的反馈直接驱动主视图的物理属性变更。
对标 LangSmith：三栏用例卡 (Input/Expected/Actual) 与性能元数据透出

在体验侧，我深度重构了实验管理与执行看板。将新建实验表单流进行标准化，透出 Latency / Tokens / Score 等核心性能元数据，并通过异常高亮、Trace Diff 下钻等设计，极大地提升了研发的“排错效率”。

02 / 数据飞轮：从生产环境到评测资产

Agent 的价值在于持续迭代。我将“线上日志观测”提炼为独立的生命周期模块，设计了从 Production Logs 到 Eval Sets 的闭环流转路径。

在工程约束上，明确了“线上 output 不能直接等同于 reference output”的铁律。为此，我们引入 Copilot 作为“标注与清洗”节点，将带有噪音的脏数据自动转化为可用的期望输出，真正建立起可持续转动的 Data Flywheel。

03 / 成本引擎：结构化与防幻觉的 Skill 落地

AI 的商业化必须回答成本问题。为了从“会说”升级到“会算、算对、可复用”，我基于底层数据资产构建了两个硬核 Skill：

Workflow Model Advisor (模型选型与降本)

建立 `pricing_config.json`（报价/积分汇率）与 `capabilities.json`（5 维能力标签）。通过编写 `model_matcher.py` 进行硬逻辑过滤与性价比计算，彻底解决了选型过程中的“盲目性、价格幻觉与能力幻觉”。最终通过 Skill 输出稳定链路下的 Top-N 平替建议报告。

Infra Cost Advisor (基建算力精算)

清洗并结构化多云厂商报价数据，编写 `infra_calculator.py` 实现 CPU/RAM/存储可传参的合并核算。输出明确的倍率、节省百分比与绝对金额，形成了“模型端 + 基建端”的双端 ROI 叙事闭环。

数据驱动 + 脚本计算 + LLM 总结：用三层解耦架构彻底压住 LLM 幻觉风险

04 / 认知沉淀：技术型 PM 的能力模型

这段经历让我完成了从纯产品思维到“工程化交付”视角的跃迁：

  • 产品推演能力：习惯于从“业务场景 → 用户任务 → 系统约束 → 交互闭环”倒推复杂后台的信息架构。
  • 工程与数据思维：深刻理解了如何用“数据结构+代码硬逻辑”框定不稳定 LLM 的边界，让 Agent 产出变得可回归、可验证。
  • 独立排障底气：能够在无文档的荒荒环境下，利用抓包与日志定位根因，补齐了边缘场景测试截面与防 AI 误改的安全存档机制。
