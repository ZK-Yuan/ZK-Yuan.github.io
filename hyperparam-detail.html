01 / 研发协同与平台 1.0 交付

在 v1.0 版本极短的迭代周期内，我深度参与并主导了 UAT 测试验收的全流程（涵盖验收标准制定、用例设计、缺陷分级、复现定位、回归验证与上线 Check）。

我利用抓包工具与后端日志联动，将系统异常精准归因到具体节点与链路环节，形成可执行的复现路径与修复建议。在此期间，成功推动修复了 30+ 项 P0/P1 级核心缺陷，包括但不限于：

  • 节点参数透传错误与变量映射异常
  • 循环节点 Schema 错位导致的数据丢失与死锁
  • Webhook 路径残缺与 Redis 变量映射错误
  • Milvus 节点能力缺失，以及文本输入互斥与 UI 交互阻塞

通过严密的工程纪律测试，最终保障了 Agent 平台编排引擎核心链路按期交付上线。

利用抓包与日志联动，进行复杂节点的数据流转溯源与诊断

02 / 自动化评测平台设计 (EvalFlow)

针对传统黑盒测试覆盖率极低的痛点，我独立负责了平台工作量评测功能的从 0 到 1，完成整体 Figma 及前端页面逻辑的交互确立，打通了“评测集管理 → 实验跑批看板 → 门禁拦截 → A/B 对撞 → Trace 下钻 → 诊断报告与修复建议”的端到端链路。

  • DAG 白盒评测方案：实现了节点级的输入/输出对比，并透出耗时、Token 等性能元数据，支持细粒度的 case-by-case diff 对比。
  • “语义映射”交互创新：引入 LLM 将生硬的技术链路转化为可读的业务场景与失败归因。直接把节点差异翻译成为什么错、错在哪、该改哪一步。

这套白盒评测体系将复杂工作流的缺陷定位排障时间从“小时级”暴力压缩至 10 分钟内，并使核心评测用例的业务验收通过率跃升至 95%。

03 / 复杂游戏 Agent 场景落地

为验证引擎在复杂状态流转与多工具编排下的稳定性与可扩展性，我主导了平台核心 Demo 工作流的策划与搭建。

基于 Redis、Python、JS、Milvus 等节点，成功落地了复杂的游戏状态智能体案例（包含具备长期记忆的好感度 NPC、以及遵循经济模型的动态交易员）。

核心架构亮点：将“记忆与状态”抽象为可插拔的工具节点。Agent 能够根据场景动态选择策略（读取 Redis 短期记忆、冷启动时不读取、长对话中仅读取摘要/关键事实），并将记忆命中情况与注入内容完整纳入 Trace 追踪，便于后期复盘与评测。

标杆案例落地：可插拔状态节点支撑面向 3 家 B 端游戏开发者的商业化演示

04 / 平台 Copilot-Skills 构建与双端成本核算

AI 的工业化落地必须回答成本问题。我搭建并设计了平台工作流的成本计算与可视化图表生成等扩展 Skills，致力于降低“成本黑盒”与“选型拍脑袋”带来的投产风险。

在模型选型与降本建议上，我设计了严谨的三层解耦架构：

  • 数据层：建立详尽的价格与能力维度配置库。
  • 脚本计算层：作为确定性引擎，按场景/复杂度/上下文/格式稳定性/速度要求，进行硬逻辑的能力过滤与性价比排序。
  • LLM 总结层：将生硬的计算数据封装为人类可读的高保真降本建议报告。

此外，我还将核算逻辑扩展至服务器、存储等基建底座，实现了模型层与基建层双管齐下的 ROI 核算，为成本看板提供了可靠的底层支撑。
